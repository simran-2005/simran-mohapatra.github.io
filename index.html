Incident Case Studies

Real problems, real resolution.

Two production incidents from DXC that show what structured troubleshooting looks like at scale.

Case 01 · Azure · 2024
Bulk VM Outage — 200+ Tickets, Microsoft Tier-3 Escalation
Problem
50–60 VMs went simultaneously unavailable, generating 200+ incident tickets. Initial assumption in the team pointed to a network failure.
Finding
Correlated all affected VMs to a single shared Azure resource group — ruled out network as the cause and identified a platform-level issue instead.
Action
Escalated telemetry-backed findings to Microsoft Tier-3 with a documented hypothesis, bypassing generic triage and accelerating the path to root cause.
Result
Datacenter-side issue identified and remediated. Environment stabilized within 2 days.
Case 02 · Windows · July 19, 2024
CrowdStrike Global Outage — Fleet-Wide Windows VM Recovery
Problem
A faulty CrowdStrike Falcon sensor update caused BSOD boot loops across affected Windows systems globally, including managed Windows VMs in the client environment.
Action
Executed Microsoft-issued ISO recovery procedures — boot from recovery media, remove the faulty driver file, restore to operational state — across affected VMs while live incident bridges were still active.
Result
Restored affected managed Windows VM services during the active incident window. Recognized with DXC Q3 Winner FY24 DXC Recognition Grow 2024.
